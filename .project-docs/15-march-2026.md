## What Was Built

### 1. Authentication Bug Fixes

**File:** `app/Http/Requests/LoginRequest.php`
- Removed `confirmed` rule from `password` field — it was requiring a `password_confirmation` field that the login form doesn't have, causing login to always fail even with correct credentials.
- Added `messages()` method with Indonesian validation messages.

**File:** `app/Http/Requests/RegisterRequest.php`
- Added `messages()` method with Indonesian validation messages (was showing English by default because Laravel's built-in messages were being used).

**Root cause of both bugs:**
- Laravel validation messages are English by default. Custom messages require a `messages()` method in FormRequest, or a `lang/id/validation.php` file.
- `confirmed` rule requires a `{field}_confirmation` input — only appropriate for register, not login.

### 2. Student Management — Full CRUD

#### Model
**File:** `app/Models/Student.php`
- Already existed. Fields: `name`, `email`, `major`, `status` (default: `active`), `enrollment_year` (year type).
- `$fillable` covers all five fields.

#### Migration
**File:** `database/migrations/2026_03_14_160924_create_students_table.php`
- `email` is unique.
- `status` defaults to `'active'`.
- `enrollment_year` uses `year` column type (integer year e.g. 2024).

#### Form Requests (generated via `php artisan make:request`)
**File:** `app/Http/Requests/StudentStoreRequest.php`
- Rules: `name`, `email` (unique:students), `major`, `status` (in:active,inactive), `enrollment_year` (integer, min:2000, max:current year).
- `authorize()` returns `true`.
- All messages in Indonesian.

**File:** `app/Http/Requests/StudentUpdateRequest.php`
- Same rules as Store, but email uniqueness ignores the current student: `unique:students,email,{$this->route('student')->id}`.
- All messages in Indonesian.

#### Controller (generated via `php artisan make:controller StudentController --resource`)
**File:** `app/Http/Controllers/StudentController.php`
- Uses Route Model Binding (`Student $student`) — not `string $id`.
- `index()`: paginate(15) with `withQueryString()`, supports search (name/email LIKE), filter by status, filter by major. Passes `$students` and `$majors` to `dashboard` view.
- `create()`: returns `students.create` view.
- `store()`: uses `StudentStoreRequest`, creates student, redirects to `students.index` with success flash.
- `show()`: returns `students.show` view (placeholder, not yet built).
- `edit()`: returns `students.edit` view.
- `update()`: uses `StudentUpdateRequest`, updates student, redirects with success flash.
- `destroy()`: deletes student, redirects with success flash.

#### Routes
**File:** `routes/web.php`
```php
Route::middleware('auth')->group(function () {
    Route::get('/dashboard', fn () => redirect()->route('students.index'))->name('dashboard');
    Route::resource('students', StudentController::class);
    Route::post('/logout', [AuthController::class, 'logout'])->name('logout');
});
```
- `/dashboard` redirects to `students.index` for backward compatibility.
- Full resource routes registered under `/students`.

## Key Conventions Established
- **Artisan first:** Always generate controllers and requests via `php artisan make:*` before modifying.
- **Indonesian messages:** All `FormRequest::messages()` methods write in Bahasa Indonesia.
- **Route Model Binding:** Controllers use `Student $student`, not `string $id`.
- **Flash messages:** Success/error via `->with('success', '...')` / `->with('error', '...')`, displayed in `layouts/app.blade.php`.
